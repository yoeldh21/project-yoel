# יועץ השקעות נדל"ן חכם

> כלי אנליטיקה מבוסס ML+GIS המסייע למשקיעים לאתר, לסנן ולבדוק עסקאות נדל"ן — מותאם אישית לפרופיל ההשקעה של כל משתמש.

---

## הבעיה שאנחנו פותרים

שוק הדיור מושפע מפרמטרים רבים ומשתנים — ביקוש והיצע, מצב השוק, אירועים פוליטיים וטבעיים. משקיע נדל"ן נדרש לנתח את כולם, לאתר עסקה משתלמת ולהחליט האם להיכנס אליה, לרוב ללא כלים מתאימים.

הכלי מרכז את המידע במקום אחד ומחזיר המלצות מדורגות בשלוש רמות זום: **אזור ← נכסים ← עסקה בודדת**.

**קהל יעד:**
- משקיעים פרטיים המעוניינים להיכנס לשוק הנדל"ן
- משקיעים מנוסים המחפשים את העסקה הבאה שלהם
- אנשי מקצוע: מתווכים ושמאים

## 5 סיפורי משתמש

**1. משקיע מתחיל — הכוונה לאזור (מצב א׳)**
כמשקיע שנכנס לשוק לראשונה, אני רוצה להגדיר פרופיל ולקבל מפה של אזורים מומלצים עם הסבר מדוע כל אחד מתאים, כדי לצמצם את מרחב החיפוש לפני שאבחן נכסים ספציפיים.

**2. משקיע מנוסה — חיפוש עסקאות באזור (מצב ב׳)**
כמשקיע שזיהה אזור מעניין, אני רוצה רשימה מדורגת של נכסים זמינים לפי התאמה לפרופיל שלי, כדי שלא אצטרך לסרוק ידנית מאות מודעות.

**3. משקיע — בדיקת עסקה ספציפית (מצב ג׳)**
כמשקיע שמצא נכס, אני רוצה להזין כתובת ומחיר ולקבל ציון כדאיות, תשואה חזויה והשוואה לעסקאות דומות — כדי להחליט האם להתקדם לבדיקת נאותות.

**4. משקיע — השוואה בין שתי עסקאות (מצב ג׳ פעמיים)**
כמשקיע שמתלבט בין שני נכסים, אני רוצה להריץ בדיקת כדאיות על שניהם ולקבל השוואה ישירה, כדי לבחור את העסקה שמתאימה ביותר למטרת ההשקעה שלי.

**5. מתווך — הצגת ניתוח מקצועי ללקוח (מצב ג׳)**
כמתווך, אני רוצה להפיק דוח כדאיות מפורט לנכס ספציפי, כדי לבסס את המלצתי על נתונים ולחזק את האמון של הלקוח בי.

## ארכיטקטורה

מנוע ML+GIS משותף לשלושת המצבים. ההבדל הוא רמת הזום בלבד.

**מודלים:** Gradient Boosting, Random Forest, XGBoost

**ספריות:** `scikit-learn` · `XGBoost` · `geopandas` · `shapely` · `geopy` · `pandas` · `numpy` · `Folium` / `Leaflet.js`

**סטק:** React (frontend) · FastAPI (backend) · PostgreSQL + PostGIS (storage)
